# 「パントリーを更新」で既存のアイテムを更新せずに重複が作成されるのはなぜ？

これは、買い物リストのアイテムと既存のパントリーアイテムが一致する **エイリアス** を共有していない場合に発生します。

**例：**
- 買い物リストに「成分無調整牛乳」がある
- パントリーに「牛乳」がある
- これらがエイリアスを共有していない場合、アプリは「牛乳」を更新する代わりに新しい「成分無調整牛乳」パントリーアイテムを作成します

**モーダルで何が起こるか確認できます：**

確認する前に、パントリー更新モーダルに2つのセクションが表示されます：
- **追加するアイテム：** 新しいパントリーアイテムとして作成されます（緑の「新規アイテム」ラベル付き）
- **更新するアイテム：** 既存のパントリーアイテムを更新します（「切れ → 在庫あり」のようなステータス変更を表示）

これらのセクションを確認して、重複が発生する前にキャッチしてください。

**重複を修正するには：**
1. パントリーから重複を削除
2. 残ったアイテムを編集して、もう一方の名前をエイリアスとして追加（例：「牛乳」アイテムに「成分無調整牛乳」を追加）

**重複を防ぐには：**
- 買い物リストとパントリーで一貫した命名を使用
- よくある表現の違いをパントリーアイテムにエイリアスとして追加
